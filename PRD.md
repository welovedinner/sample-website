# 聚餐小助手 - 产品需求文档

## 核心定位

把群聊里散落的聚餐意向，压缩成几条清晰的"选项"，让新来的人不用翻完所有聊天记录就能快速了解有哪些聚餐机会。

---

## 要做 ✅

| 功能 | 说明 |
|------|------|
| 展示当前聚餐选项 | 时间、地点、发起人一目了然 |
| 简单的报名/表达意向 | 让用户快速加入感兴趣的活动 |
| 信息聚合看板 | 把漫长的聊天压缩成几条类似选项的东西 |
| 快速了解情况 | 进来就能看懂，不用翻几百条消息 |

---

## 不需要做 ❌

| 功能 | 原因 |
|------|------|
| 复杂的日程投票 | 现有界面刚好，不需要过度设计 |
| 群聊/沟通功能 | 沟通在微信群里就能完成 |
| AI 智能匹配 | 初期用户量不需要，大家应该 figure out 自己的需求 |
| 餐厅推荐系统 | 超出核心需求范围 |

---

## 产品形态

简洁的**信息看板**，类似精简版公告栏：

```
📋 当前聚餐选项

1️⃣ 除夕包饺子 @kyou老师家
   📅 2/16 下午  📍 15号线附近  👥 3人已加入
   
2️⃣ 年后寿喜烧
   📅 待定  📍 待定  👥 2人感兴趣
   
3️⃣ 元宵节聚餐
   📅 2/24  📍 待定  👥 1人发起
```

---

## 设计原则

1. **简洁优先** - 不做不需要的功能
2. **降低门槛** - 进来就能看懂
3. **群聊互补** - 网站展示信息，沟通留在群里
